    算数比较

  • [ 2 -eq 2 ] 相等
  • [ 2 -ne 2 ] 不等
  • [ 3 -gt 2 ] 大于
  • [ 2 -ge 2 ] 大于等于
  • [ 1 -lt 2 ] 小于
  • [ 2 -le 2 ] 小于等于

字符串比较

  • [str1=str2] 相同的话$?结果为真
  • [-n str] 字符串不是空结果为真
  • [ -z str] 字符串为空结果为真
  • [[“xxxx”==x*]] 在表达式中表示0或者多个字符
  • [[xxx==x??]]在表达式中表示单个字符
  • 在引用变量的时候记得加双引号[-z “$a”]否则当a未定义的时候语法会报错

逻辑判断

  • [ 2 -ge 1 -a 3 -ge 4] -a and
  • [ 2 -ge 1 -o 3 -ge 4] -o or
  • [[ 2 -ge 1 && 3 -ge 4]] 逻辑与
  • [[ 2 -ge 1 || 3 -ge 4]] 逻辑或
  • [ ! 2 -ge 1 ] 逻辑非

内置判断

  • [ -e file ] 如果文件存在，则结果为真
  • [ -d file ] 如果文件是一个子目录，则结果为真
  • [ -f file ] 如果文件是一个普通文件，则结果为真
  • [ -r file ] 如果文件可读，则结果为真
  • [ -s file ] 如果文件长度不为0，则结果为真
  • [ -w file ] 如果文件可写，则结果为真
  • [ -x file ] 如果文件可执行，则结果为真
  • [[]]是[]的扩展语法，在老的sh里并不支持，推荐用[]

常见的组合命令linux三剑客

grep：基于正则表达式查找满足条件的行

  • grep pattern file
  • grep -i pattern file 忽略大小写
  • grep -v pattern file 不显示匹配的行
  • grep -o pattern file 吧每个匹配的内容用独立的行显示
  • grep -E pattern file 使用扩展正则表达式
  • grep -A -B-C pattern file 打印命中数据的上上下文
  • grep pattern -f dir / 递归搜索
  • pattern正则表达式
  • 演练 （curl http://www.baidu.com 打印网页内容）

awk： 根据定位到的数据处理其中的分段

awk理论上可以代替grep
awk ‘pattern{action}
awk ‘BEGIN{}END{}’ 开始和结束
awk ‘/Running/‘ 正则匹配
awk ‘/aa/,/bb/‘ 区间选择
awk ‘$2~/xxx/‘ 字段匹配
awk ‘NR==2’ 取第二行
awk ‘NR>1’ 去掉第一行

awk的字段数据处理 (一行一行处理)

  • -F 参数指定字段分隔符
  • RS 记录行分隔符，吧指定符号变为\n. ORS 可以看成RS的逆向过程。吧\n换成指定符号
  • FS 字段分隔符，FS默认值为空格，以指定符号，把字符串分成多个字段。OFS输出的字段分隔符，将字段间加入符号
  • NF 字段数 NR 记录数
  • BEGIN{FS=”_”}也可以表示分隔符
  • BEGIN{RS=”_”}以_分割字符串
  • $0代表当前的记录
  • $1代表第一个字段
  • $N 代表第N个字段
  • $NF 代表最后一个字段

    sed：根据定位到的数据行处理其中的分段

  • sed [addr]X[options]
  • -e 表达式
    sed -n ‘2p’ 打印第二行
  • sed ‘s#hello#world#’ 修改
  • -i 直接修改源文件
  • -E扩展表达式
  • –debug 调试
  • sed ‘s///g’ 加g会全局查找替换，不加g只替换第一个符合条件的字符串

bash运行环境

shell运行环境概念

  • bash下还可以在重新启动一个shell，这个shell是sub shell，原shell会复制自身给他。在sub sehll 中定义的变量，会随着sub sehll 的消亡而消失
  • （）子shell’中运行
  • $(ls)表示执行ls后的结果，与``作用一直，不过可以嵌套
  • {}当前shell中运行
  • $$当前脚本执行的pid
  • &后台执行
  • $!运行在后台的最后一个作业的pid（进程id）

    shell环境变量

  • set 可以获得当前的所有变量
  • unset 可以释放变量
  • env 可以获得可以传递给子进程的变量
  • export aa=bbb 把私有变量导出，等价于declare -x

传参

  • $0表示执行的程序，是相对于执行目录的路径
  • $1$2$3分别表示第几个参数，默认shell只支持9个参数，使用shift可以传递更多参数，
  • $@,$* 表示所有的参数，不含$0
  • $来表示多个参数
  • $@可以在函数之间传递参数，并不改变参数的排列
    • 比如123’45“67”8’。这个参数在经过多次传递后，依然表示4个参数
    • 级的使用”$@”，不要直接使用$@
  • $* 会把参数打散，相对顺序不变，但是参数的个数就变成8个了。
  • 演示方法ff()
    ff(){ for d in $@”$@”$*”$*”;do echo $d;done;};ff 12’3”45”6’7”89”

执行方式

  • chmod u+x xxx.sh;./xxx.sh
  • bash xxx.sh(这种方式会开启一个sub shell)
  • source xxx.sh (当前shell中执行 .xxx.sh同义)
  • eval 可以执行shell原意语句
  • exec 尽量不要使用，这会破坏当前的shell

debug

  • bash -x (读取每一行，并执行，可以方便的看到执行的语句)
  • ser -x 在当前shell中调试，set +x还原
  • trap 相关的信号，包括ERR,DEBUG等
  • trap ‘cmd’ ERR
