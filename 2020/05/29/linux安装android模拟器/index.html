安装 sdk

  • 基础包:http://tools.android-studio.org/index.php/sdk 选择 linux 版本下载(wget url)

  • 解压缩: tar zxvf tgz 包

  • 解压完成后，执行 sdk 更新下载命令

  • tools/android update sdk –no-ui 下载的太多不用执行(更新所有 sdk)

  • 因为模拟器需要有 system.img 文件，才能创建出来

  • 所以我们需要先通过 tools/android list sdk –extended –no-ui –all 查询一下有哪些 sys-img*文件,我们过滤一下./android list sdk --extended --no-ui --all | grep sys-img

  • ./android update sdk -a --no-ui --filter sys-img-armeabi-v7a-android-23

  • 如果报缺少依赖 sdk，执行./android list sdk -a找到需要的 sdk 安装./android update sdk -u -t 46 -a

  • android list target查看有哪些 sdk

  • 创建模拟器：android create avd -n 模拟器名字 -t 上一步的id序号 (如: android create avd -n mytest1 -t 18),系统会询问用户是否需要定制硬件配臵清单(Do you wish to create a custom hardware profile),这里选择 no ,使用缺省的硬件配置

  • 这样创建出来的内存等参数配置特别差，我们需要改一下硬件配置参数。android create avd -n stf1 -t 17 -s 480x800 -c 是存储空间

  • 创建好后，通过android list avd 来查看已经创建了哪些模拟器

  • 可能碰到如下错误：

    • android-sdk-linux/tools/qemu/linux-x86_64/qemu-system-armel: error while loading shared libraries: libpulse.so.0: cannot open shared object file: No such file or directory碰到如上错误，输入 cat /etc/redhat-release，查看当前 linux 系统版本为 CentOS Linux release 7.4.1708 (Core) 。到https://pkgs.org/download/libpulse.so.0 安装对应的版本使用命令行：yum install pulseaudio-libs
    • -bash: ./emulator: /lib/ld-linux.so.2: bad ELF interpreter: No such file or directory执行yum install glibc.i686
    • error while loading shared libraries: libstdc++.so.6: cannot open shared object file: No such file or directory。执行命令： yum whatprovides libstdc++.so.6然后会提示哪个安装包。然后执行yum install libstdc++-4.8.5-39.el7.i686
    • PANIC: HOME is defined but could not find stf1.ini file in $HOME/.android/avd(Note: avd is searched in the order of $ANDROID_AVD_HOME,$ANDROID_SDK_HOME/.android/avd and $HOME/.android/avd)。sudo cp -r /root/.android /home/用户名/

  • ./emulator -avd stf1 -no-window -no-boot-anim -no-audio -gpu off

nohup emulator -avd stf1 -no-window -no-boot-anim -no-audio -gpu off >log.txt 2>&1 &
