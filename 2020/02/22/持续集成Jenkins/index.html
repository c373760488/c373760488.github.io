安装

创建job

  • 首页 create job
  • Discard old builds 构建记录按天或者条数保存
  • This project is parameterized 参数化项目
  • Build Triggers
    • Build periodically 构建时间，5个*分别代表分钟、小时、天数、月数、星期数
      H/2 * * * * 两分钟一次
      H/2 * * * 1-5 周一到周五没两分钟
  • Build 构建代码，可以选择Windows batch command，shell
    • 选择shell echo “hellow $strparam”
  • save
  • Build with Parameters
  • Build
  • Build History 选一个点进去，选择Console Output查看执行结果

ssh连接需要插件ssh slave

配置用户,设置权限

设置权限需要插件：Role-based Authorization Strategy

配置email

  • 系统配置
    • E-mail Notification
      • SMTP server : smtp.163.com
      • Default user E-mail suffix : @163.com
      • 高级 - Use SMTP Authentication,账号密码，Use SSL，SMTP port， Reply-To Address(接受回复的邮箱)
    • Jenkins Location
      • Jenkins URL:http://localhost:8080/jenkins
      • 系统管理员邮件地址:xxx@qq.com

        构建触发邮件

  • 创建项目最下面 构建后操作添加 Editable Email Notification
  • Advanced Setting

参数化构建

创建项目-参数化构建过程-添加参数

父子任务

  • 创建job1
  • 创建job2，构建触发器-其他工程构建后触发。填写job1，选择触发条件
    • 稳定时触发：成功时触发
    • 即使不稳定时触发：稳定，不稳定都触发。 安装text finder，可以再构建后操作添加post action设置当检测到某字符串时，执行结果变为不稳定的，
    • 即使失败时触发：成功，不稳定，失败都会触发

单元测试(git job)

  • 创建job
  • 源码管理选择Multiple SCMs再选择git，填写Repository URL， Additional Behaviours新增检出到子目录，填写目录名（意思是把git的文件down在这里）
  • 添加执行shell
  • 添加构建后操作-publish JUint test result report（需要JUnit插件）测试报告填写为*/.xml 代表当前目录创建
  • publish html report 可以添加生成的html报告，需要添加groovy postbuild，不然js代码不能运行

jenkins api

  • build
  • 获取最新任务编号
  • 查询任务状态

jenkins web hook

  • jenkins系统配置-GitHub-高级-覆盖 Hook URL-为 Github 指定另外一个 Hook URL。
  • 把url复制到github项目settings-webhooks下
  • jenkins项目中打开GitHub hook trigger for GITScm polling

pipeline

  • Declarative pipeline
  • Scripts pipeline

  • 创建pipeline script项目
    创建项目，选择流水线(pipeline)，创建后选择pipeline script,填写脚本保存运行。。

  • 创建pipeline script from scm

    • scm 选择git，添加地址https://github.com/c373760488/iPipeline.git
    • 脚本路径直接写git里的文件名就好了
