做一个性能监控平台

所需工具

  • docker
  • JMeter：压测⼯具， 产⽣压测数据
  • InfluxDB: 开源时序数据库， 特别适合⽤于处理和分析资源监控
    数据，⽤于存储压测数据
  • Grafana：度量分析与可视化图标展示⼯具，可以⽀持不⽤种类的
    数据源，⽤于将存储于InfluxDB中的数据以图表的形式展示出来

环境部署

  • influxdb

    下载influxDB镜像： docker pull influxdb
    docker run -d -p 8086:8086 -p 8083:8083 –name=jmeterdb influxdb
    docker exec -it jmeterdb bash

  • jmeter

    jmeter中在Backend Listener implementation 中选择
    InfluxdbBackendListenerClient （注意：必须使⽤jmeter 5.0以上版本，
    否则需要单独装插件才可以看到这个组件）
    在influxdbURL中将实际的influxdb hostname填写进去
    在application中填写 order
    在testTitle中填写 Order Testing
    其余配置保持不变

  • grafana
    docker pull grafana/grafana
    docker run -d -p 3001:3000 –name=jmeterGraf grafana/grafana

jmeter详解

jmeter八大元件说明

  • Sampler(取样器)：不与其他元件发生交互的作用的元件
  • Logic Controller()支队子节点的sampler有效，二其他元件需要与sampler等元件交互
  • Config Elements(配置元件)：影响其他范围内的所有元件
  • Pre-processors(前置处理器)：在其作用范围内的每一sampler元件之前执行
  • Timer(定时器)：对其作用范围内的每一个sampler有效
  • Post-processors(后置处理器)：在其作用范围内的每一个sampler元件之后执行
  • Assertions(断言)：对其作用范围内的每一个sampler元件执行后的结果执行校验
  • Listener(监听器)：手机器作用范围内的每一个sampler元件的信息并且呈现出来

jmeter八大元件 作用于以及上下级关系

Sampler独立，因此不存在作用域问题
Logic Controller仅对其子节点中的sampler生效
其他元件，如果是sampler的子节点，则仅对其父节点作用
除sampler和Logic Controller外的其他元件，如果其父节点不是sampler，作用域是该元件父节点下的其他所有后代节点（包括子节点，子节点的子节点等）
想听节点加载/执行顺序：
Config Elements
pre-processors
Timer
Sampler
Post-processors
Assertions
Listener
