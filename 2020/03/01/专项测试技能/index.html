专项测试技能

发表于 2020-03-01 更新于 2020-03-05

和用户体验强相关的app性能指标

  • 启动时间：47%用户希望启动时间<2s
  • 流畅度：16.67ms绘制一帧作为极限流畅度
  • 稳定性：不出现app崩溃访问错误等
  • 页面加载：5s后，每增加1s，会降低7%的访问量

    专项测试（技术维度）

  • 崩溃
    • 自动遍历，monkey测试，横竖屏切换，快速进退
  • 卡顿（掉帧，gc，cpu）
    • 卡顿测试，内存泄漏测试，menthod profile
  • 响应慢（启动时间，交互相应，h5加载
    • 冷热启动，界面切换，h5性能测试）
  • 发热（cpu,men,io,network,gps等硬件使用）
    • method profile,gc统计，io统计，流量统计，硬件使用统计，耗电量分析
  • 兼容性测试（机型覆盖，回归）
    • 兼容性测试，自动化测试，自动遍历，monkey测试

      常用测试方案

  • Android
    • 自己编写sdk
    • ddms
    • android studio最新版本的集成工具
    • hook
    • 代码插桩
  • iOS
    • instruments
    • hook
    • 代码插桩

崩溃测试

崩溃问题类型

典型问题

  • ANR;
    • 主线程5s内没响应
  • NullPotinterException
    • 空指针
  • ClassCastException
    • 类型转换异常
  • Force Close：
    • 未捕获的android VM 异常
  • Crash:
    • 未处理的native异常
      崩溃检测通过 adb log :S:E 捕获异常

      基本测试方法

  • 发布前
    • monkey测试—+appcrawler自动遍历
    • 结合各类场景用例
  • 发布后
    • 埋点
    • 接入外部sdk

异常场景（常见）

  • 后台接口问题：
    • 弱网：完全超时。2G，3G
    • 接口返回异常：null返回
    • 接口变更问题：字段类型变更
  • 逻辑问题
    • 异步线程问题：打开新闻页在快速按返回
    • 逻辑处理不当：横竖屏切换，前台后台切换
  • 内存问题：低内存，循环翻页，执行可累计内存的操作

    实战

    思考

  • 什么时间点做性能测试
    • 第一轮功能测试结束，无重大修改
  • 基于什么基础测上测
    • 历史版本&竞品
  • 什么时候性能优化
    • 一般情况只有在大版本迭代或者重点性能优化才进行
  • 要哪些场景
    • app核心场景，必须优化场景
  • 要测什么包
    • 一般侧正式包（release），反应真实用户体感
  • 要测几轮
    • 一般测正式一轮，剩余看开发优化

性能专项测试报告

App冷启动

  • adb shel am force-stop com.qiyi.video
  • adb shell am start -S -W com.qiyi.video/.WelcomeActivity
  • adb logcat | grep -i activitymanager.*Displayed
  • lazy load:只能埋点
  • this Time 最后一个activity时间
  • totalTime：包括简介启动的activity时间
  • waitTime：总体小耗时间

    录屏

  • adb shell screenrecord –time-limit 10 /sdcard/video.mp4
  • adb pull /sdcard/video.mp4 /Users/liyu/Desktop/video
  • 视频分帧
    • ffmpeg -i d:/test/video.mp4 -r 100 -threads 2 d:/test/Android-Capture-%05d.png

      硬件使用信息

  • adb shell dumpsys cpuinfo
  • adb shell dumpsys meminfo packagename or pid
  • adb shell getprop dalvik.vm.heapgrowthlimit 查看进程可用的最大内存
  • adb shell cat /proc/#pid#/net/dev 流量数据获取 飞行模式可以清零流量
    • receive表示收包
    • transmit比搜啊还是发包
    • bytes表示收发的字节数
    • packets表示手法正确的包量
    • errs表示收发错误的包量
    • drop表示收发丢弃的包量

关键硬件资源的占用率

  • adb shell dumpsys procstasts –hours 3
  • adb shell dumpsys meminfo package_name | pid [-d]
  • adb shell dumpsys batterystats –charged package-name
  • adb shell dumpsys netstats detail
  • adb shell dumpsys gfxinfo package-name

    卡顿分析

    卡顿影响因素：
  • 内存问题：（内存抖动，full gc）
  • cpu （计算耗时）
  • render（布局复杂，overdraw）

    使用systrace

    路径：/Users/coye/Library/Android/sdk/platform-tools/systrace
    使用：python2 ./systrace.py运行，然后操作app，完成后按enter结束，打开生成的html
    找到对应的app 红色F代表绘制有延迟，绿色正常。鼠标选中一片区域查看slices
  • python2 ./systrace.py -l列举所有支持的性能数据分析
  • python2 ./systrace.py -t 10手机10s左右的系统性能数据
  • python2 ./systrace.py -t 10 -j保存一份json数据文件用于编程分析
